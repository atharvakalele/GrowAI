# Feeds API v2021-06-30 — Official Reference
**Source:** Amazon GitHub OpenAPI Spec
**Our Endpoint:** sellingpartnerapi-eu.amazon.com
**IMPORTANT:** JSON feeds ONLY (XML deprecated per our project rules)

## Endpoints (6 operations)

### 1. List Feeds
- **Path:** GET /feeds/2021-06-30/feeds
- **Rate:** 0.0222 req/sec (burst: 10)

**Parameters:**
| Name | Type | Required | Description |
|------|------|----------|-------------|
| feedTypes | array (1-10) | No | Feed type filter |
| marketplaceIds | array (1-10) | No | Marketplace filter |
| pageSize | integer (1-100) | No | Default 10 |
| processingStatuses | array | No | CANCELLED, DONE, FATAL, IN_PROGRESS, IN_QUEUE |
| createdSince | datetime | No | Default 90 days ago |
| createdUntil | datetime | No | Default now |
| nextToken | string | No | Pagination |

### 2. Create Feed
- **Path:** POST /feeds/2021-06-30/feeds
- **Rate:** 0.0083 req/sec (burst: 15)

**Request:**
```json
{
  "feedType": "JSON_LISTINGS_FEED",
  "marketplaceIds": ["A21TJRUUN4KGV"],
  "inputFeedDocumentId": "from-createFeedDocument"
}
```
**Response 202:** { "feedId": "string" }

### 3. Get Feed
- **Path:** GET /feeds/2021-06-30/feeds/{feedId}
- **Rate:** 2 req/sec (burst: 15)
**Response:** Feed object with processingStatus + resultFeedDocumentId

### 4. Cancel Feed
- **Path:** DELETE /feeds/2021-06-30/feeds/{feedId}
- **Rate:** 2 req/sec (burst: 15)
Only works on IN_QUEUE feeds.

### 5. Create Feed Document (Upload URL)
- **Path:** POST /feeds/2021-06-30/documents
- **Rate:** 0.5 req/sec (burst: 15)

**Request:**
```json
{ "contentType": "application/json; charset=UTF-8" }
```
**Response 201:** 
```json
{ "feedDocumentId": "string", "url": "presigned-upload-url (expires 5 min)" }
```

### 6. Get Feed Document (Download URL)
- **Path:** GET /feeds/2021-06-30/documents/{feedDocumentId}
- **Rate:** 0.0222 req/sec (burst: 10)
**Response:** { feedDocumentId, url (presigned download), compressionAlgorithm: GZIP }

## Feed Processing Flow:
```
1. createFeedDocument → get upload URL
2. PUT your JSON data to upload URL
3. createFeed → get feedId
4. Poll getFeed until processingStatus = DONE
5. getFeedDocument with resultFeedDocumentId → get result
6. Download result → check for errors per item
```

## Feed Types (JSON) We Need:
- JSON_LISTINGS_FEED — create/update listings (title, price, inventory)
- POST_FLAT_FILE_PRICEANDQUANTITYONLY_UPDATE_DATA — price + quantity updates

## Processing Status:
- IN_QUEUE → IN_PROGRESS → DONE (success) or FATAL (error)
- CANCELLED (if cancelled before processing)

## Key Capabilities:
- **Bulk Price Update:** JSON feed with price changes for 100s of ASINs
- **Bulk Inventory Update:** quantity changes
- **Bulk Listing Update:** title, bullet points, descriptions

## CRITICAL: 
- ⚠️ ALL WRITE OPERATIONS — NEEDS MSIR APPROVAL
- Upload URL expires in 5 minutes
- Result may show partial success (some items fail, others succeed)
- JSON format ONLY (our rule: no XML)

## Error Codes: 400, 401, 403, 404, 413, 415, 429, 500, 503
